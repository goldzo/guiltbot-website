# Moderation
- purge [amount]: Purges a specified amount of messages from the channel.
  - Optional Argument: [amount] - Specifies the number of messages to purge. If not provided, the default behavior might be to purge all messages.
  
- lock (channel): Locks the specified channel.
  - Required Argument: (channel) - Specifies the channel to be locked. Without specifying a channel, the command cannot determine which channel to lock.

- unlock (channel): Unlocks the specified channel.
  - Required Argument: (channel) - Specifies the channel to be unlocked. Without specifying a channel, the command cannot determine which channel to unlock.

- nuke: Clones and deletes the current channel.
  - No arguments required. This command operates on the current channel where it's executed.

- ban (user) [time]: Bans a user, optionally specifying a duration.
  - Required Argument: (user) - Specifies the user to be banned.
  - Optional Argument: [time] - Specifies the duration of the ban. If not provided, the ban might be indefinite.

- warn (user) [reason]: Warns a user, optionally providing a reason.
  - Required Argument: (user) - Specifies the user to be warned.
  - Optional Argument: [reason] - Provides a reason for the warning. If not provided, the warning might be issued without a specific explanation.

- roles [user]: Displays the roles of a specified user.
  - Optional Argument: [user] - Specifies the user whose roles are to be displayed. If not provided, the command might default to displaying the roles of the user who executed the command.

- ticket [subcmd]: Opens a ticket for support or assistance.
  - Optional Argument: [subcmd] - Specifies a subcommand or additional options for opening a ticket. The necessity of this argument depends on the specific implementation of the ticket system.

- close: Closes a previously opened ticket.
  - No arguments required. This command operates on the current ticket being closed.
